## Major Projects

### 1. QodlyCoder (Final Year Project - PFE)
**Duration:** February 2025 - June 2025  
**Technology Stack:** Python, LangGraph, RAG, LLMs, React, TypeScript, Qwen-3-32B, Llama-3.3-70B

**Description:**  
An innovative AI-driven tool specifically engineered for generating sophisticated React-based Qodly Custom Components with comprehensive TypeScript support, utilizing advanced Large Language Models enhanced with contextual data systematically extracted from Qodly platform documentation and relevant UI library resources.

**Key Features:**
- **Agentic Workflow:** Multi-agent system orchestrated by LangGraph managing code generation, validation, and file creation
- **Hybrid RAG Pipeline:** Combines semantic understanding with lexical precision through dual-phase architecture
- **Tool-Augmented Agent:** Primary agent with decision-making capabilities determining when external tools (RAG, web search) are required
- **Conditional Routing:** Feedback loop with rule-based validation ensuring code compliance (maximum 3 iterations)
- **CLI Interface:** Automated retrieval, generation, and file editing from natural language prompts

**Technical Implementation:**

*Preprocessing Phase:*
- Advanced parsing extracting semantic units (interfaces, types, classes, enums)
- Context enhancement using GPT-4.1 mini for generating JSDoc descriptions
- Dual indexing strategy: semantic pathway (vector database) and lexical pathway (BM25)

*Runtime Phase:*
- Parallel semantic and lexical query processing
- Score fusion with empirically optimized weights
- Cross-encoder reranking for final refinement

*Web Search Integration:*
- Intelligent search engine with Crawl4AI (max_depth=2, max_pages=15)
- BFSDeepCrawlStrategy for comprehensive documentation coverage
- Focused summaries limited to 2000 characters with 150-word summaries

**Prompt Engineering Techniques:**
- Instruction-based prompting for React functional component generation
- Few-shot prompting for Qodly-specific file transformations
- Structured schemas ensuring compliance with @ws-ui/webform-editor interface

**Generated Files:**
- main.tsx, App.tsx, index.css
- ExampleComponent.build.tsx, ExampleComponent.render.tsx
- ExampleComponent.settings.tsx, ExampleComponent.config.tsx

**Results:**
- 28.9% improvement in retrieval performance over baseline
- Successful automation of component generation across varying complexity levels
- Significant reduction in manual coding workload
- Enhanced framework compatibility and integration

**Challenges Addressed:**
- Complex multi-file organizational structures
- Framework-specific conventions and design patterns
- TypeScript type safety and code reliability
- Seamless integration with Qodly Studio's drag-and-drop interface

**Future Directions:**
- Fine-tuning Qwen2.5-Coder-7B-Instruct using LoRA/QLoRA techniques
- Integration of reinforcement learning framework for continuous improvement
- Enhanced handling of complex custom requirements with minimal human intervention
